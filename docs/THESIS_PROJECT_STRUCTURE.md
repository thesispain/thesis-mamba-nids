## 1. Core Notebooks (The "Brain")

These two notebooks contain the entire experimental pipeline.

### `Part1_SSL_Pretraining.ipynb`
**Purpose:** Self-Supervised Learning (SSL) pipeline. Pre-trains models on unlabeled benign traffic to learn robust feature representations before any supervised fine-tuning.

*   **1.1 - 1.2:** Imports & Path Setup. Configures GPU and defines paths to `weights/ssl`, `data/unswnb15_full`.
*   **1.3 - 1.4:** Data Loading & Exploration. Loads `pretrain_50pct_benign.pkl`. Visualizes feature distributions (Protocol, LogLength, Flags, etc.). Computes normalization stats.
*   **1.5:** Supervised Data Load. Loads `finetune_mixed.pkl` (only for later validation, not used in training).
*   **1.6 - 1.9:** **Model Architecture Definitions.**
    *   `Mamba`: Pure PyTorch implementation of the Selective State Space Model.
    *   `PacketEmbedder`: Tokenizes raw 5-tuple packets.
    *   `BiMambaEncoder`: Bidirectional teacher model (Paper Contribution).
    *   `BertEncoder`: Transformer baseline (for comparison).
*   **1.10 - 1.11:** **SSL Augmentation & Loss.**
    *   Defines `CutMix` and `Anti-Shortcut Masking` (Contribution 1).
    *   Defines `NT-Xent` contrastive loss + Reconstruction auxiliary loss.
*   **1.12:** **Training Loop.** Runs the pre-training for BiMamba and BERT. Saves weights to `weights/ssl/`.
*   **1.13:** Analysis. t-SNE plots of learned embeddings.

### `Part2_Evaluation_Testing.ipynb`
**Purpose:** Supervised Fine-Tuning, Distillation (TED), and Final Evaluation.

*   **2.1 - 2.2:** Setup & Imports.
*   **2.3:** **Model Definitions.** (Repeats definitions from Part 1 for standalone execution, adds `UniMambaEncoder` and `BlockwiseStudent`).
*   **2.4:** Data Loading. Splits `finetune_mixed.pkl` into Train (70%) and Test (30%). Implements Few-Shot splitting.
*   **2.5:** Utils. `train_classifier`, `evaluate_model` functions.
*   **2.6:** **Distillation (TED).** Defines the Student training loop with "Temporally-Emphasised Distillation" loss (Contribution 3).
*   **2.7:** **Teacher Fine-Tuning.** Loads SSL weights from Part 1 and fine-tunes them on labeled data.
    *   Trains: `BiMamba+Masking`, `BiMamba+CutMix`, `BiMamba_Scratch`, `BERT+Masking`.
*   **2.8:** **ML Baselines.** Trains XGBoost and Random Forest for comparison.
*   **2.9:** **Student Training.** Trains the lightweight `UniMamba` student using TED from the BiMamba teacher.
*   **2.10 - 2.12:** **Evaluation & Ablation.**
    *   **Early Exit:** Tests dynamic exit thresholds (confidence vs accuracy).
    *   **Cross-Dataset:** Evaluates on CTU-13 and CIC-IDS (Zero-shot transfer).
    *   **Latency:** Measures inference speed (ms/flow).

## 2. Directory Breakdown

### `scripts/`
Contains standalone Python scripts for batch execution.
*   **`full_crossdataset_eval.py`**: A CLI version of the cross-dataset evaluation logic. It loads pre-trained weights and runs anomaly detection (AUC/F1) on UNSW, CTU-13, and CIC-IDS. Useful for running long evals without a notebook.

### `results/`
Stores raw experimental data (JSON/TXT) generated by the notebooks.
*   **`full_pipeline_results.json`**: Aggregate metrics for all models.
*   **`crossdataset_full_results.json`**: Detailed cross-dataset AUC/F1 scores.
*   **`early_exit_results_v2.json`**: Statistics on how many packets the student model processes before exiting.
*   **`q2_*.json`**: Results specific to "Question 2" (Architecture/Latency analysis).

### `docs/`
narrative documentation and reports.
*   **`THESIS_HERO_REPORT.md`**: The high-level executive summary of the thesis.
*   **`CHAPTER_5_RESULTS.md`**: The fully written "Results" chapter for your thesis document.
*   **`FACULTY_PRESENTATION.md`**: Draft slides/notes for your defense.
*   **`THESIS_CONTEXT.md`**: A "memory file" tracking the current project state, key decisions, and next steps.

### `weights/`
Stores trained model checkpoints (`.pth` files).
*   **`ssl/`**: Self-supervised pre-trained weights (e.g., `bimamba_masking_v2.pth`). **Critical for Part 2.**
*   **`teachers/`**: Fine-tuned teacher models (e.g., `teacher_bimamba_masking.pth`).
*   **`students/`**: Distilled student models.

### `data/`
*   **`unswnb15_full/`**: Contains the main UNSW-NB15 pickle files (`pretrain_50pct_benign.pkl`, `finetune_mixed.pkl`).
*   **`ctu13_flows.pkl`**, **`cicids2017_flows.pkl`**: External datasets for generalization testing.

## 4. Special Info / "Gotchas"

*   **OOM Errors:** If `Part1` crashes, reduce `num_workers=0` in the DataLoader. The dataset is large.
*   **Dependency:** `Part2` *requires* `Part1` to be run first (or pre-trained weights to be present in `weights/ssl`).
*   **Mamba Import:** The code uses a **pure PyTorch implementation** of Mamba (defined in the notebooks) rather than the CUDA kernel version (`mamba_ssm`). This ensures it runs on any GPU/CPU without complex compilation, but might be slightly slower than the optimized kernel.

This structure is clean and "thesis-ready". The separation of SSL (Part 1) and Evaluation (Part 2) is a standard academic practice.
